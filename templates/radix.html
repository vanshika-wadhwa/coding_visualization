Radix Sort

Radix sort is a non-comparative integer sorting algorithm. It processes the digits of numbers and sorts them from the least significant digit (LSD) to the most significant digit (MSD), or vice versa. This sorting technique is efficient when the range of input numbers is small.

Description

  • Stable Sort: Radix sort is stable, meaning that it preserves the relative order of equal elements.
  • Digit-by-Digit Sorting: It sorts elements by processing each digit of the numbers in a specified order (LSD or MSD).
  • Efficient for Small Range: Radix sort is particularly effective when dealing with numbers that have a small range of digits.

Example

Consider sorting the array [170, 45, 75, 90, 802, 24, 2, 66] using radix sort:

  1. Step 1: Sort by the least significant digit (LSD). After the first pass:
  2. [170, 802, 2, 24, 45, 75, 90, 66]
  3. Step 2: Sort by the second least significant digit. After the second pass:
  4. [170, 802, 2, 24, 45, 75, 66, 90]
  5. Step 3: Sort by the most significant digit. After the third pass, the array becomes sorted:
  6. [2, 24, 45, 66, 75, 90, 170, 802]

Characteristics

  • Time Complexity: Radix sort has a time complexity of O(nk), where n is the number of elements and k is the number of digits in the largest number.
  • Space Complexity: O(n + k), where n is the number of elements and k is the range of digits.
  • Stable: Radix sort is stable and ensures that elements with the same digit value maintain their relative order.
  • Non-Comparative: Unlike algorithms such as quicksort or mergesort, radix sort does not compare elements directly.

Radix sort is particularly useful when sorting large datasets of integers, especially when the numbers have a limited number of digits. It is not typically used for floating-point numbers or strings.
